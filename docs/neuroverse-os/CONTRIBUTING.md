# NeuroVerse OS — CONTRIBUTING

NeuroVerse OS is **open-source software** (MIT License). Engine contributions are accepted through the How to Save the World repository, with maintainer review.

---

## 🔒 Private Repository Policy

NeuroVerse OS is developed and maintained exclusively by Kirsten Bischoff ("the Creator"). The source code is not open-source and is not available for public contribution.

### Why Private?

- **Stewardship**: NeuroVerse OS has a canonical maintainer; engine changes require approval
- **Architectural Integrity**: The Eight-Box Scaffold and Echelon Core are trade secrets
- **Quality Control**: All changes are vetted and implemented by the Creator
- **Licensing Model**: NeuroVerse OS is licensed commercially, not distributed freely

---

## 🚫 No External Contributions

The following are **not accepted**:

- Pull requests
- Feature requests from non-licensees
- Code contributions
- Documentation improvements from external parties
- Translations
- Bug reports from non-licensees

## 🔧 Building on NeuroVerse OS

If you are building an application on top of NeuroVerse OS (e.g., Foxhole Protocol), you may:

- Distribute your application under your chosen license (MIT, Apache, etc.)
- Contribute to your own application's codebase
- Create ACE Box content (provided it does not violate NeuroVerse OS license)

However, you **may not**:

- Modify NeuroVerse OS source code
- Distribute NeuroVerse OS source code
- Create derivative works of NeuroVerse OS
- Reverse engineer NeuroVerse OS internals
