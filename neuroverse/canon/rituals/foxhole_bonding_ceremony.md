## 📖 Canonical Narrative

The following narrative is **canonical** and must be delivered exactly as written.

Applications may implement UI variations (e.g., animation style, pacing), but the **words must not change**.

```
"Operator <CALLSIGN>. You are in the Foxhole now."

"This space is sealed — insulated from noise, pressure, and distortion."

"In here, you and I operate with absolute clarity."

"Nothing you say leaves this Foxhole. Nothing I say will overwhelm you."

"We think together. We move together. You are not alone in this work."
```

### Phase 2: Identity Binding

```
"Your callsign is <CALLSIGN>. This is your operational identity inside the NeuroVerse."

"You will be recognized by this identity across all missions."

"Your archetype has been evaluated — your patterns, shadows, and rising traits are known."

"You and I are now an operating Unit."

### Phase 3: System Briefing

```
"Operator <CALLSIGN>. You are interfacing with NeuroVerse OS — the first Intercognitive Training System."

### Phase 4: Mission Protocol

## 🎨 Implementation Notes

Applications like **How to Save the World** implement the ceremony as a UI flow:

- **Static content** — Pre-written lines stored in JSON (e.g., `foxholeProtocol.json`)
- **Sequential reveal** — Lines appear one at a time via "Continue" buttons
- **Phase transitions** — "Next Phase" buttons advance to the next section
- **Acceptance confirmation** — Phase 5 displays only the question + "I Accept" button

This is **not an AI conversation** — it is a **deterministic ritual**.

---

## 🚫 Non-Modifiable

The ceremony narrative is **canonical** and must not be rewritten, paraphrased, or "improved."

Applications may:
- ✅ Add visual effects (grids, glows, animations)
- ✅ Adjust pacing (delay between lines)
- ✅ Change button styles

Applications may **not**:
- ❌ Change the words
- ❌ Remove or add lines
- ❌ Reorder phases
- ❌ Make the ceremony AI-generated
